# VRCX Insights Tool

`VRCX Insights Tool` 是一个给普通用户使用的本地桌面工具，用来分析你自己的 `VRCX.sqlite3`，帮助你回看在 VRChat 里的互动关系、共同活动轨迹，以及和某些人的共处情况。

它的定位很简单：把 VRCX 已经记录下来的历史数据，整理成更容易看的结果。

## 它能做什么

你可以用它来查看：

- 你最常遇到的陌生人或熟人
- 你和某个人在什么时间、什么世界里待过
- 哪些人和你长期同房、共同出现频率高
- 某个好友和其他人的关系排行
- 两个人之间的共处明细、重叠时间段、房主信息、峰值人数等

## 适合谁

这个工具适合：

- 想回看自己 VRChat 活动记录的用户
- 想分析长期互动对象的用户
- 想从 VRCX 历史里整理社交轨迹的用户

如果你只是想打开后直接看结果，而不是折腾数据库或代码，这份 README 就是按这个使用场景写的。

## 数据安全吗

是的，核心分析是本地完成的。

- 工具读取的是你本机上的 `VRCX.sqlite3`
- 程序默认以只读方式打开数据库
- 不会修改你的 VRCX 数据库内容
- 程序只会在自己的配置目录保存一个很小的 `data.json`，用来记住你选过的 VRCX 数据文件夹

补充说明：

- 当某些记录里缺少世界名称时，程序可能会请求 VRChat 的 world API 来补全世界显示名
- 这个请求只用于补全世界名字，不会上传你的数据库文件
- 如果网络不可用，核心分析仍然可以使用，只是部分世界可能显示原始 world id 或 location

## 你需要准备什么

使用前只需要准备两样东西：

- 一份可用的 `VRCX.sqlite3`
- Windows 打包版，或者 Node.js 环境

如果你是普通用户，优先推荐直接使用 Windows 打包产物。

## 怎么启动

### 方式一：直接使用 Windows 打包版

项目支持以下 Windows 产物：

- 安装版
- Portable 便携版

如果你使用的是打包版：

- 不需要额外安装 Node.js
- 不需要安装 `sqlite3` 命令行
- 启动后按界面提示选择 VRCX 数据文件夹即可

### 方式二：从源码运行

如果你是自己运行源码，可以使用：

```bash
cd vrcx-insights-tool
npm install
npm start
```

如果你想在启动时直接指定数据库路径，可以这样运行：

```bash
cd vrcx-insights-tool
VRCX_DB_PATH=../VRCX.sqlite3 npm start
```

## 第一次打开后怎么用

第一次启动时，程序会先尝试帮你找到 VRCX 数据目录。

- Windows 下会优先尝试自动识别 `%APPDATA%/VRCX`
- 如果没有识别到，界面会提示你手动选择 VRCX 数据文件夹
- 选择完成后，程序会记住这个目录，下次通常不需要重复设置

正常使用流程如下：

1. 启动应用
2. 选择或确认 VRCX 数据文件夹
3. 等待首页读取数据库信息
4. 通过顶部时间范围筛选你想分析的区间
5. 在不同页面中查看结果

## 常见问题

### 1. 为什么打开后没有数据？

通常是以下几种情况：

- 没有选对 VRCX 数据文件夹
- 目标目录下没有 `VRCX.sqlite3`
- 你的 VRCX 历史记录本身较少
- 当前时间筛选范围太短

可以先尝试：

- 在“设置”里重新选择数据目录
- 把时间范围改成“一年”或“有史以来”
- 执行一次手动刷新

### 2. 为什么有些世界名字显示不完整？

可能原因：

- 本地记录里本来就没有完整世界名
- 网络不可用，无法补全世界名称
- VRChat 接口返回失败

这种情况下不影响主要分析结果，只是显示名称可能不够友好。

### 3. 会不会改坏我的 VRCX 数据？

正常情况下不会。

这个工具按只读方式访问数据库，设计目标就是“只分析，不写入”。

## 已知限制

- 结果依赖 VRCX 历史记录本身是否完整
- 如果你的数据库缺少某些关键表，程序可能无法正常初始化
- 世界名称补全依赖网络；网络不可用时不影响核心本地分析，但名称可能不完整
- 当前项目主要提供 Windows 打包流程；macOS / Linux 更适合源码运行
